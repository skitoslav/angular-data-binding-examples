Description

Based in service strategy is one of the most common approaches to working with data in Angular.
Basically all manipulations with data are moved to Service to make Component's class smaller and easier to understand.

This example provides only the simplest implementation of keeping data's related functionality in Service.
There are hundreds ways how to organize a Service's structure.
I guess it happens because Service and DI provide a lot of space for developers (may be even to much =] )

Code example


        export class YourComponent {
          constructor(public readonly userService: UserService) { }
        }

        export class YourUsersService {
          users$ = this.list$();

          constructor(private readonly httpClient: HttpClient) { }

          list(): Observable<IUser[]> {
            return this.httpClient.get<IIUser[]>I('./api/users');
          }
        }

Recommendation

It is a very basic example. Not recommended in production
